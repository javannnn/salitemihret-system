# Finance Admin Guide

Document status: Client finance operations guide  
Version: 1.0

## Finance Admin Responsibilities
Finance Admins manage payment entry, correction workflows, finance reports, contribution tracking, and reconciliation.

## Payment Entry
1. Open Payments.
2. Select the member, household, school, sponsorship, or relevant service context.
3. Enter amount, currency, method, service type, date, and reference details.
4. Save the payment.
5. Confirm the ledger entry appears with the correct details.

## Ledger Integrity
The payment ledger is append-only. Finance users must not edit historical records directly. If an error is found, use the correction workflow so the original entry remains available for audit.

## Corrections
1. Open the original payment.
2. Start correction.
3. Enter corrected amount/details and reason.
4. Submit the correction.
5. Verify the original is linked to the correction.

## Reconciliation
Recommended reconciliation checks:

- Daily totals by payment method.
- Monthly totals by service type.
- Member contribution summaries.
- Sponsorship received vs. pledged amounts.
- School fee totals.
- Exported report totals against payment ledger filters.

## Reports
Finance Admins should use reports for:

- Daily payment summary.
- Monthly contribution summary.
- Sponsorship payment reconciliation.
- School fee reporting where configured.
- Export files for accounting review.

## Finance Controls
- Use individual user accounts only.
- Do not share finance credentials.
- Record correction reasons clearly.
- Keep receipt references consistent.
- Escalate unexplained discrepancies immediately.
