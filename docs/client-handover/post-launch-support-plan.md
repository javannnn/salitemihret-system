## Included Support
- Bug fixes for agreed in-scope delivered functionality.
- Production issue investigation.
- Access and role configuration guidance.
- Report/export troubleshooting.
- Data correction guidance where system behavior is involved.
- Minor documentation updates.
- UAT-approved deferred fixes.

## Not Included Without Change Request
- New modules.
- Major workflow changes.
- New integrations.
- New public website redesign or major content restructuring.
- Data cleanup unrelated to migration or system defect.
- Custom reports outside agreed scope.

## Priority Levels
| Priority | Definition | Response Target |
| --- | --- | --- |
| P1 Critical | Production down, security issue, data loss | Urgent / same business day |
| P2 High | Major workflow blocked | 1 business day |
| P3 Medium | Workaround available | 2-3 business days |
| P4 Low | Minor issue or question | Scheduled |

## Escalation Path
1. User reports issue to support desk.
2. Client support lead validates priority.
3. Delivery team investigates if in support period.
4. Fix or workaround is communicated.
5. Issue is logged in maintenance log.
6. Change request is created if outside scope.

## Support Review
Support issues should be reviewed weekly during the initial post-launch period and monthly or quarterly thereafter.
