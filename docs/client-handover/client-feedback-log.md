# Client Feedback Log

Document status: Stakeholder feedback tracking template

## Purpose
This log records client feedback during UAT and after go-live. Feedback may result in defect fixes, usability adjustments, training updates, documentation updates, or formal change requests.

## Category Guidance
| Category | Meaning |
| --- | --- |
| Defect | Expected agreed behavior is not working |
| Enhancement | Improvement outside the current tested behavior |
| Training | User needs additional instruction or workflow explanation |
| Documentation | Manual, guide, or support article needs update |
| Change Request | Out-of-scope or contract-impacting change requiring approval |

## Review Cadence
- During UAT: reviewed daily during defect triage.
- First 2 weeks after go-live: reviewed at least twice weekly.
- Support period: reviewed according to the support plan.
