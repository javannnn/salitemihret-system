# Admin Manual

Document status: Client administrator guide  
Version: 1.0

## Admin Responsibilities
Administrators are responsible for managing users, roles, permissions, module access, reports, operational settings, and go-live support coordination.

## User Management
Super Admins can:

- Create or invite users.
- Assign roles.
- Link users to member records where appropriate.
- Suspend, unsuspend, delete, or restore users according to client governance.
- Generate password resets.
- Review account and role audit activity.

## Role Management
The role manager controls module visibility and permission flags. Before assigning a role, verify:

- Which modules the role can see.
- Whether the role can view, create, edit, delete, approve, export, or manage records.
- Whether field-level restrictions are needed.
- Whether hidden supporting access is required for workflows like sponsorship member lookup.

## Membership Administration
Admins and authorized PR users manage:

- Member records.
- Household relationships.
- Import validation.
- Priests, ministries, tags, and controlled reference data.
- Archive and recovery workflows.

## Payments Administration
Finance admins manage:

- Ledger entries.
- Payment corrections.
- Daily close or lock behavior where enabled.
- Financial exports and reconciliation.
- Payment-related report access.

## Reports Administration
Admins can verify:

- Which reports are available by module.
- Which roles can access each report.
- Whether exports match client reporting needs.
- Whether report issues are logged for follow-up.

## Email Administration
Super Admins can access the admin email area where enabled. Email settings and credentials must be managed through the approved secure configuration process.

## Media Administration
Media Admins or Kahen users approve or reject media requests. Approved content is intended to publish to the public feed or website-facing content area.

## Operational Controls
Administrators should regularly review:

- Active users and assigned roles.
- Suspended or inactive users.
- Failed login trends.
- Backup status.
- Open support issues.
- Pending UAT/post-go-live feedback.
