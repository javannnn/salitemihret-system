# Knowledge Base and Support Desk Guide

Document status: Client support guide  
Version: 1.0

## Support Process
Users should report support requests through the client-designated support channel.

Required information:

- Name and role.
- Module or page.
- Record ID or member name if applicable.
- What the user was trying to do.
- What happened instead.
- Screenshot if available.
- Urgency and business impact.

## Issue Categories
| Category | Examples |
| --- | --- |
| Access | Login problem, missing module, wrong role |
| Data | Incorrect member/payment/sponsorship/school record |
| Workflow | Cannot complete intake, payment, correction, approval, conversion |
| Report | Missing report data, export issue, reconciliation issue |
| Website | Public website content, bilingual display, public media |
| Training | User needs guidance |
| Change Request | New feature or scope change |

## Response Targets
| Priority | Definition | Target Response |
| --- | --- | --- |
| P1 Critical | System down, data loss, security issue | Same business day / urgent escalation |
| P2 High | Major module blocked | 1 business day |
| P3 Medium | Workaround available | 2-3 business days |
| P4 Low | Cosmetic, training, minor request | Scheduled |

## Escalation
Escalate immediately for:

- Suspected security issue.
- Incorrect financial totals.
- Production outage.
- Data loss or accidental deletion.
- Critical go-live blocker.
