## Severity Definitions
| Severity | Definition | Required Action |
| --- | --- | --- |
| Critical | Data loss, security exposure, system unavailable, or core operation impossible | Immediate fix before go-live |
| High | Major module workflow blocked with no acceptable workaround | Fix before go-live or formal client deferral |
| Medium | Workflow affected but workaround exists | Fix in UAT or post-go-live backlog with approval |
| Low | Cosmetic, wording, alignment, minor usability issue | May be scheduled after go-live |

## Status Values
Open, In Progress, Ready for Retest, Retested Pass, Retested Fail, Deferred by Client, Closed, Not a Defect.

## Resolution Notes
Each resolved item must include:

- Root cause summary.
- Files or configuration changed, if applicable.
- Deployment version or commit.
- Retest evidence.
- Client confirmation for closure or deferral.
